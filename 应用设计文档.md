# CHUNITHM 定数时光机 · 应用设计文档

> 一句话：**拖动时间，看任意一天的"定数地图"长什么样。**
> 解决玩家痛点：旧视频/标题信息滞后于定数改动、玩家脑海中的定数体系固化、难以"考古"历史定数。

---

## 1. 核心理念

时间轴是主控制器，其余一切随它实时变化。用户不是在查"现在的定数表"，而是在**穿越回任意一天，看当时的定数表**。
- 拖到 2018 年 → 看到的就是当时玩家口中的"14 級"。
- 拖到 VERSE 上线日 → 一眼看清 VERSE 大改动了哪些谱。
- 拖过某歌下架日 → 它从列表消失。

## 2. 数据基础（已就绪）

- `tools/data_final.json`：**1911 首**（现行 + 下架，排除 WE/未収録/污染），**1884 首有完整逐代历史**。
- 每首 schema：
  ```
  title / status(active|delisted) / releaseDate / endDate(下架日,可空)
  version(初出代) / wiki_url
  const {expert, master, ultima}          ← 当前定数
  charts {EXP, MAS, ULT: {timeline:[{gen, genName, date, const}]}}  ← 逐代变迁
  ```
- 定数只在换代时变；"某日定数" = 时间线里 `date ≤ 选中日` 的最后一条。
- 已知边界：13 首"定数不明"(早期下架曲,小数定数遗失)；14 首无 wiki 页(仅有当前定数)；4 首 const 有 0.1 微差(待复核)。

## 4. 各区详解

### ① 时间轴
- 21 代做成**彩色色带**（每代一色，宽度≈实际时长），代名标在带上；拖动=在代际间穿梭。
- 点某条色带=跳该代；**代际吸附**开关让滑块只停在代边界（定数只在换代变，停代内无意义）。
- **播放键**：日期从 2015 自动推进到现在，看定数版图被"挤压/扩张/重排"——VERSE 那种大改最直观。
- 支持手动输入日期、◀▶逐日微调、一键跳今天。

### ② 状态条
实时显示：选中日期、所属代、当天已实装曲目数、当前筛选命中数。

### ④ 谱面列表（核心视图）
- **一行一首歌**，列显示该日期下 EXP/MAS/ULT 的定数（从时间线查 `date ≤ 选中日` 的末值）。
- **变迁轨迹**：迷你箭头串（14.0→14.1→15.3→15.5），高亮当前所处段。
- **变动标记**（停在换代日时）：本代有变动的行打 ▲升(绿)/▼降(蓝)/★新追加——"这一代改了啥"一目了然。
- 定数按标级着色（如 15+ 更深色），沿用 CHUNITHM 习惯。
- 虚拟滚动，1900+ 行丝滑。

### ⑤ 详情面板
- 该曲 21 代完整时间线（每难度一条横轴：代×定数的小折线图）。
- 追加日 / 下架日 / 初出代、wiki 链接、配信終了标记。
- 玩家术语标注（如"旧15"=VERSE 前即为 15 的谱）。

### 进阶功能（按优先级排）
- **只看本代变动**：列表只留本代有变动的谱——考古精华。
- **代际对比模式**：选两个代，差异视图显示"这两代间变了什么"（一键回答"VERSE 改了哪些"）。
- **播放动画**：时间自动推进，看版图演变。

## 5. 特殊数据处理

- **可用窗口**：歌曲只在 `[releaseDate, endDate]` 内出现；拖过下架日即消失（开"显示已下架"则灰显+划线）。
- **定数不明**（13 首）：列表里该格显示"不明"或灰显，详情页说明"早期数据遗失"。
- **WE**：整个排除（无定数）。WE 谱下架不影响常规曲状态（已在数据层处理）。

## 6. 技术方案（建议）

- **纯静态 SPA**：SvelteKit 或 React+Vite；`data_final.json` 随站打包/内嵌。
- **虚拟滚动**列表（如 @tanstack/virtual）。
- **全前端、无后端** → 本地双击即跑、可单文件分享、可原样传服务器（满足"本地优先+可分享+未来可上服务器"）。
- 难度配色沿用官方（EXP 绿 / MAS 紫 / ULT 红），代际色带自定。
- 移动端响应式（基础可用）。

## 7. 待定设计分叉（请你拍板）

1. **行粒度**：一行一首歌（难度做列，推荐）vs 一行一张谱（song×difficulty）？
2. **默认焦点难度**：默认主看 MASTER（多数玩家最关心），EXP/ULT 作可切换列？还是三难度等权？
3. **进阶功能优先级**："只看本代变动" / 代际对比 / 播放动画——v1 最想要哪个先做？
4. **视觉语言**：贴近 CHUNITHM 官方风格 vs 独立简洁风？

## 8. 建议开发里程碑

- **M1骨架**：静态站 + 时间轴 + 列表(虚拟滚动) + 基础筛选 → 能拖动看任意日定数快照。
- **M2详情**：点行看完整代际时间线图 + wiki 链接。
- **M3考古**：只看本代变动 + 变动标记(▲▼★) + 代际对比。
- **M4打磨**：播放动画、移动端、视觉精修、单文件打包分享。
